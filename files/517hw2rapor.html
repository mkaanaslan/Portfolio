Solving the Median Tour Problem using Tabu Search
1) Introduction
Median tour problem is the one of the most encountered optimization problems in the
operations research. The problem consists of n customer locations and 1 depot location.
A tour must be found includes p locations to select and customers on the other non-
selected locations should travel to their nearest node. The objective is the minimizing the
total tour length and the total customer travel distance. Although the problem could be
modeled as an integer problem, it is solved with the search algorithms in this assignment.
There are 3 different location sets consist of 50, 75 and 100 customer locations are given.
Hill climbing algorithm is used as a base algorithm to find an initial minimum objective value
for all the customer sets. Hill climbing algorithm starts from a random initial solution and
find a local optimum from it. Then a tabu search algorithm is constructed and applied to the
same random initial solution to find a better solution than the hill climbing algorithm.
2) Tabu Search Algorithm
Tabu search is a metaheuristic that uses neighborhood search algorithm to find efficient
optimal solutions. Most of the neighborhood algorithms have the problem that sticking on
the local optimum could be solved with the tabu search. Algorithm overcomes this problem
with allowing the worsening solutions and memory structures. Tabu search constrains the
search space with short term memory. Short term memory means banning some move
operations to find different neighboring solutions. In addition, there is a long-term memory
that makes the algorithm to search different neighborhood spaces with adding a penalty to
the previous search spaces.
In this assignment, solutions are hold as a vector with p elements and there is a
function for calculate the objective value from the solution vector. Also, there is a function
that finds all possible neighboring solutions from a solution. 1 element from the solution
vector is interchanged with a node which is not on the solution vector and this move is
called a swap. Neighboring solutions are found with 1-flip on the main solution which means
swapping 1 element from the solution. Tabu search is applied to the swap operations to find
an optimal solution. The pseudo code for the Tabu Search algorithm is shown below:
Tabu Search for Median Tour Problem:
Input: X solution vector, Distance Matrix, Max Iterations, Seed
Output: Optimal solution vector
Begin:
1) Find an initial random solution and assign it to the Best Solution and Current
Solution.
2) Assign the objective value of the random initial solution to the Best Objective and
Current Objective.
3) Create Tabu Time and set Tabu Iteration Counter and Iteration Counter to 0.
4) Initiate Tabu Swaps and Long Memory lists and also, set Penalty K to maximum
value of the Distance Matrix.
5) Repeat until the Iteration Counter=Max Iterations:
5.1) Find all neighboring solutions of the Current Solution as Neighbor List
5.2) Set a very high value for the Min Neighbor objective.
5.3) Repeat for all the Neighbor in the Neighbor List:
5.3.1) Find the SWAP move to reach this Neighbor from the Current Solution.
5.3.2) Find the FREQUENCY of the SWAP move from the Long Memory list.
5.3.3) Calculate the objective value of the Neighbor solution.
5.3.4) IF objective value plus FREQUENCY times Penalty K is less than the Min
Neighbor:
5.3.4.1) IF SWAP move is not in the Tabu Swaps or the objective value is less
than the Best Objective:
5.4.1.1) Assing objective value to Min Neighbor.
5.4.1.2) Assign Neighbor to Best Neighbor
5.4) Find the SWAP move to reach Best Neighbor from the Current Solution
5.5) Add SWAP move to Tabu Swaps and Long Memory list.
5.6) Assign Best Neighbor to Current Solution.
5.7) Assign Min Neighbor to Current Objective.
5.8) IF Current Objective is less than Best Objective:
5.8.1) Assign Current Solution to Best Solution.
5.8.2) Assign Current Objective to Best Objective
5.9) Increase Tabu Iteration Counter by 1.
5.10) IF Tabu Iteration Counter equals to Tabu Time:
5.10.1) Delete first SWAP element from the Tabu Swaps.
5.10.2) Set Tabu Time.
5.11) Increase Iteration Counter by 1.
6) Return Best Solution.
END;
