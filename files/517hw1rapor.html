1) Introduction
Assignment optimization problems are very common in operations research area. There
are couple different types of assignment problems. While some of them are not difficult to
solve, some others are NP-hard problems. The main topic of this assignment is a NP-hard
problem: the Quadratic Assignment Problem (QAP). Since this problem could not solved in
polynomial time, it will be solved by using a constructive and an improvement heuristic
algorithm. The Quadratic Assignment Problem is modeled mathematically as the following:

multiplication of the 2 decision variables in the objective function, this is a non-linear
optimization problem, however there is some ways to transform it to a linear problem which
is not the topic of this assignment. The problem is solved with 2 different heuristics and the
results are recorded.
2) Algorithms
The 2 algorithms are coded in the R language. Primarily, since these are heuristics
methods, an objective value calculation function is needed. Coding the objective function
calculator with the 4 iterators (i,j,k,l) and 4 loops as the objective function in the model is
very inefficient. Instead, the objective function is calculated by manipulating the flow matrix
then multiplying it with the distance function. The flow matrix manipulation is done
according to the decision variable X vector. X is not in the matrix form, it used as named
vector form in the code part. Names(X) is the vector which indicates the location numbers
and elements of the X vector indicates the facility numbers. For example, first element of
the X vector is 5, this means facility 5 is assigned to location 1. Here is the pseudo code for
the objective calculation:
Objective Calculator Fast:
Input: X vector, Distance Matrix, Flow Matrix
Output: Objective value according to X vector
Begin:
1) Assign n <- row number of Distance Matrix
2) Create a copy matrix from the Flow Matrix
3) names(X) is the location vector, X values are the facility vector
4) Repeat for i =1 to n;
4.1) Assign Column number Facility[i] from Flow Matrix to Column number Location[i] of
Copy Matrix
5) Assign Copy Matrix to Temp Matrix
6) Repeat for i =1 to n;
4.1) Assign Row number Facility[i] from Temp Matrix to Row number Location[i] of Copy
Matrix
7) Multiply Copy Matrix and Distance Matrix elementwise as Result Matrix
8) Return the sum of every instance of the Result Matrix
End;
The objective value is calculated with 4 loops it took 3.72 seconds while it took 0.01 seconds
with the new objective function calculator. Since the time constraint is solved, heuristic
algorithms can be built.
2.1) Construction Algorithm
Construction algorithms are very useful to find an initial solution, and they find it
efficiently. However, these algorithms generally find a solution a bit far from the optimal
solution. Construction algorithm for the QAP primarily finds the location with the minimum total
distance to all locations and the facility with the maximum total flow to all other facilities. Then
it finds the next location that has a minimum distance with the initial location and add it to the
list of locations and again it finds the next location that has the minimum distance with all
locations in the list of locations until list has all locations. The same process applied to the
facilities too, but it finds the facilities with the maximum flow. Finally, it assigns the facilities in
the facility list to the locations in the location list with the same order and returns the solution X
vector. Here is the pseudo code for the construction algorithm:
Construction Algorithm:
Input: Distance Matrix, Flow Matrix
Output: Solution vector X
Begin;
1) Create 2 lists, location and facility, initiate n = row number of Distance Matrix
2) Find the location which has the minimum total distance to all other locations and add to
facility list
3) Finde the facility which has the maximum total flow with all other facilities and add to
facility list
4) Repeat until location list has length of n:
4.1) Find the location not in the location list which has the minimum distance with all
locations in the location list and add to the location list.
5) Repeat until facility list has length of n:
5.1) Find the facility not in the location list which has the maximum flow with all facilities
in the facility list and add to the facility list.
6) Assign facilities to the locations in the same order and record it on the X vector, then
return X vector.
End;
2.2) Improvement Algorithm
Improvement algorithms starts at an initial solution and find a better solution.
Improvement algorithms finds better solutions than the constructive algorithms and generally
they find solutions near the optimal solution. For the QAP, improvement algorithm takes an
initial solution X vector as input and also flow and distance matrices. Then it computes every
possible pairwise change on the X vector with objective calculation function and make the
pairwise swap with the minimum objective value. Algorithm make this in one iteration, and it
computes iterations until the minimum objective value does not change, then it returns to the
minimum objective value and its solution vector X. Here is the pseudo code for the
improvement algorithm:
Improvement Algorithm:
Input: Initial Solution X vector, Distance Matrix, Flow Matrix
Output: Minimum objective value and final solution X vector
Begin;
1) Compute the objective value with the initial solution vector X and assign it to a min
variable. k is iteration count, and it starts with 0 value.
2) Repeat until mink+1=mink:
2.1) Change X vector with every possible pairwise facility swipes and compute the
objective value.
2.2) If any swipe has a less or equal objective value than mink, assign mink+1 that value.
2.3) Assign Xk+1 the new objective solution.
3) Return mink+1 and Xk+1
End;
3) Conclusion and Results
Both construction algorithm and the improvement algorithm applied to the QAP with
n=30, n=40 and n=50. Construction algorithm finds solution with a gap approximately 20% from
the optimal solution for all the n values. Improvement algorithm finds a solution with gap 3-4%
from the optimal solution if it has the input from the construction algorithm. When random
initial solutions are inserted to the improvement algorithm, it finds solutions with different gap
for different n values. Gaps are along 2% for the n=30, along 5% for the n=40 and along %20 for
the n=50 values. Finally, while computation time of the construction algorithm is 0.24 or 0.25
seconds for every n value, computation time is different for n values with improvement
algorithm. It took about 5-6 seconds for the n=30, about 10-12 seconds for the n=40 and about
23-25 seconds for the n=50.
