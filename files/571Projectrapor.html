Solving the Traveling Salesman Problem with Profits using Ant Colony
Optimization and Simulated Annealing
1) Introduction
Travelling salesman problem is one of the well-known optimization problems and it is
hard to solve. Therefore, problem is used as a benchmark for solution algorithms. Travelling
salesman problem with profits (TSPP) is another kind of the problem where the objective is
maximizing the total profit which is the customer profits minus the tour cost. There are n
customers with different profit levels and the goal is finding the best tour where the total
profit is maximum. In this assignment, 6 different customer sets with n values 51,76,101
are provided and there are 2 sets of customers low profile (LP) and high profile (HP) for
every n value. Ant colony optimization and simulated annealing search algorithms are used
to solve the problem.
2) Ant Colony Optimization
When the problem is a traveling salesman problem, the first algorithm come to mind is
Ant Colony Optimization (ACO). The algorithm is inspired by the ants in the nature and their
pheromone paths. In this algorithm arbitrary ant population construct solutions with
probabilities in respect to pheromone paths. Then these pheromones are updated according
to the objective values of the solutions. There are 6 different parameters for this algorithm:
T0 initial pheromone trails value, m ant population size, r evaporation coefficient, Q
pheromone update constant, e best solution constant and MAX is the maximum number
of iterations. In the experiments with eil51-LP data set, a good solution is couldnt find
with that algorithm with default parameters, a fine tuning should be made. Here is the
pseudo code for the Ant Colony Algorithm:
Ant Colony Optimization for Travelling Salesman Problem with Profits:
Input: Customer demand vector, distance matrix of customers , t0,m,r,Q,e,MAX
Output: Best solution and best objective
Begin:
1) Create Visibility Matrix as 1/Distance Matrix, create T Matrix same as Distance
Matrix then fill it with t0 value.
2) Set N Customers as length demand vector (0 included), set Customer List as customer
nodes (0 excluded), set Best Objective as very small number. (-99999)
3) Repeat for 1 to MAX:
4.1) Create Population data frame, and fill with m solutions, solution and objective value
as columns.
4.2) For m times:
4.2.1) Set space as Customer List and create solution vector, set i=0.
4.2.2) While length(space)>0:
4.2.2.1) Randomly select a node in space, assign to j.
4.2.2.2) Value = (T Matrix(i,j)*Visibility Matrix(i,j)*Demand[j])/
SUM(T Matrix[j,][space]*Visibility Matrix[j,][space]*demand[space])
4.2.2.3) Create random number p between 0,1, and IF p<Value:
4.2.2.4) i=j, add j to solution vector.
4.2.3) Add solution and solution objective to Population as a row.
4.2) IF the solution with biggest solution objective in Population > Best Objective:
4.2.1) Assign solution to Best Solution and its objective value to Best Objective.
4.3) Create Delta Matrix and Delta Best as N CustomerxN Customer matrices.
4.4) For every solution in the population:
4.4.1) Add the 0 node in the beginning and end of the solution.
4.4.2) For every (i,j) path in the solution:
4.4.2.1) delta = Q/|Q-Solution Objective|
4.4.2.2) Delta Matrix(i,j) = Delta Matrix(i,j) + delta
4.4.2.3) IF Solution Objective=Best Objective:
4.4.2.3.1) Delta Best(i,j) = Delta Best(i,j) + delta
4.5) For every (i,j) in T Matrix:
4.5.1) T Matrix(i,j) = T Matrix(i,j) + Delta Matrix(i,j) + e*Delta Best(i,j)
5) Return Best Solution and Best Objective.
END;
3) Simulated Annealing
Simulated annealing is a meta-heuristic algorithm which is inspired by the physics of the
annealing process. This algorithm searches random solutions with parameters. It has 5
different parameters: t initial temperature, r cooling ratio, ip temperature parameter,
L random neighbor iteration and fp another temperature parameter. These parameters
are also should be tuned and after the tuning it gave good solutions. Here is the pseudo
code:

4) Conclusion and Results
The travelling salesman problem with profits is solved with both Ant Colony
Optimization and Simulated Annealing. Experiments are made on eil51-LP data set and
unfortunately Ant Colony Optimization didnt give good solutions, because it should be
tuned. However, algorithm takes too much time to run and therefore it couldnt used to
solve the problem with all the datasets. Simulated Annealing was easy to implement and
easy to tune since it doesnt take too much time to run. Problems with all the data sets are
