Solving the Generalized Assignment Problem using Genetic Algorithm
1) Introduction
Generalized Assignment Problem is one of the most common NP-complete problem
encountered in the operations research. The objective is minimizing the total cost of the
assignments between m agents and n jobs. Every job must be assigned to exactly one
agent. Every agent has a resource capacity, and the jobs are consuming the resources of the
agent they are assigned to. Therefore, resource capacity of the agents cant be exceeded to
find a feasible solution. There are 4 different problem settings with different m and n
values are given, and the goal in this assignment is finding the best solution with the meta-
heuristic search algorithm Genetic Algorithm.
2) Genetic Algorithm
Genetic Algorithm is a metaheuristic algorithm that finds best solutions in an efficient
way. It is inspired by the Charles Darwins evolution theory. In this assignment Genetic
Algorithm is used for solving the Generalized Assignment Problem according to Chu and
Beasley (1996). Solutions are constructed as vectors in length of the number of jobs. Agents
are the elements or genes of the solution. For 4 different GAP problems there are 4 different
cost matrices, consumption matrices and resource vectors. In the algorithm, primarily a
population of size N is created with random solutions. For every solution there are 2
performance values which are the fitness and the unfitness according to the paper. In every
iteration, a child is generated with crossover, mutation and (a)/(b) improvements steps.
Then child is replaced with a low-performance individual from the population. The iterations
are continued for M consecutive steps that there is not any child solution generated that
improves the solution. Here is the pseudo code for the Genetic Algorithm:
Genetic Algorithm for Generalized Assignment Problem:
Input: N population size, M stopping condition value, X solution vector, Cost Matrix,
Consumption Matrix, Resources Vector, Seed
Output: Final Population Data Frame
Begin:
1) Set the Seed value and create the empty data frame Population with 3 columns
Fitness, Unfitness and X.
2) Create N random solutions and append to Population data frames X column with fitness
and unfitness values.
3) Set Child Count to 0 and initialize Best Solution to the solution with lowest unfitness and
fitness values in that order from the Population data frame.
4) Repeat until the Child Count=M:
4.1) Generate Child:
4.1.1) Select 4 random solutions from the Population and select the solution with the
lowest fitness value among the first 2, call it Parent 1, then the last 2 and call it
Parent 2. This means select 2 parent solutions with 2 Binary Tournaments.
4.1.2) Create random number P among 1 to length of the parent solutions. Then select
a parent solution randomly. Finally select the first P genes from the selected parent,
select the genes from P to length of the parent, then merge the genes in a vector and
assign it to Child variable. This is the cross-over step.
4.1.3) Select 2 random genes from the Child and swap these genes. This step is called
the mutation.
4.1.4) For every Agent in the Child solution, calculate the total consumed resource of
the Agent. If the total resource consumption exceeds the total resource of the
Agent; select a random job from the jobs that assigned to the Agent then assign to
the first next Agent that has available resources and update the Child.
4.1.5) For every Job in the Child solution, search another Agent that has available
resources. If the assignment of new Agent to that Job has a lower objective value
then the previous assignment, assign that Job to new Agent and update the Child.
4.1.6) If the Child solution found in the previous steps, has a duplicate in the current
Population, go back to step 4.1.1. Else create the Child solution.
4.2) Replace the solution with the highest Unfitness value with the Child solution. If all
the solutions have the 0 Unfitness value, then replace the solution with the highest
Fitness value.
4.3) Check if the Best Solution with the lowest fitness value is changed, if it is changed
assign the new solution to the Best Solution and set Child Counter to 0. If it is not
changed increase the Child Counter by 1.
5) Return the Population data frame.
END;
3) Conclusion and Results
The genetic algorithm is applied for the 4 problems which has different agent and job
settings. For every problem, algorithm starts with an initial population with size 1000 and
algorithms are run until 1000 consecutive steps without generating a child solution that
improves the solution. The best solutions are approximately 3% away from the optimal
solution. Average objective values of the populations are away from the optimal solution by
5% to 10% depending on the problem settings. Increment of the number of jobs causes the
